Guia básico - Git

Configurando um usuário

git config --global user.name "Seu Nome"

git config --global user.email seu.email@email.com

Verificar as informações

git config --list

git config --global -e

Criando um repositório

Inicializa um novo repositório Git localmente no diretório atual.

git init

Clona um repositorio remoto já existente, para o diretório local.

git clone <URL>

quando usar um servidor remoto, seu comando será

git clone usuário@servidor:/caminho/para/o/repositório

Adicionar & Confirmar

Esse comando irá adicionar todos os arquivos novos e/ou modificados a Staging .

git add.

Esse comando irá adicionar um arquivo em específico a Staging .

git add <Nome do arquivo>

Mover um arquivo da Staging para o Repositório local

git commit -m "comentários das alterações"

Observe seu repositório

Exibe o estado atual do repositório, incluindo arquivos modificados, adicionados e confirmados.

git status

Mostra as linhas exatas que foram adicionadas ou removidas.

git diff

Exibe o histórico de commits.

git log

Enviando alterações

Mover um arquivo do repositório local para, o repositório remoto

git push

Ramificando

Para criar uma nova branch em um repositório Git, você pode usar o comando

git checkout -b minha-nova-feature

Para retornar para a branch master, você pode usar o comando

git checkout master

Para remover uma branch no Git, você pode utilizar o comando

git branch -d minha-nova-feature

Para tornar uma branch disponível para outros colaboradores de um repositório Git remoto, você precisa primeiro garantir que a branch tenha sido enviada para o repositório remoto.

git push origin minha-nova-feature

Atualizar & Mesclar

Atualizar e mesclar um repositório Git envolve normalmente duas operações principais:

obter as últimas atualizações de um repositório remoto

git pull

mesclar essas atualizações com a sua branch local.

git merge <nome_da_branch>
